# 安全策略

## 支持版本

当前仅最新版本（main 分支）接受安全修复。
不维护旧版本的安全补丁。

## 报告安全漏洞

如果您发现了安全漏洞，**请不要**公开提交 Issue。

请通过 GitHub 的[安全公告](https://github.com/bin3413532696-dev/Table/security/advisories)功能提交私密报告，内容包括：

- 漏洞类型和影响范围
- 复现步骤（或 PoC）
- 受影响的版本
- 可能的修复建议（可选）

## 响应流程

1. **48 小时内**：确认收到报告并启动评估
2. **7 天内**：完成影响评估并确定修复计划
3. **修复后**：发布安全更新并通过 GitHub 安全公告披露

## 安全注意事项

本项目为个人工作台，部署时请注意：

- **生产环境必须** 替换 `PROVIDER_SECRET_KEY` 为强随机密钥（>= 32 位）
- **生产环境必须** 设置 `ALLOW_DEFAULT_USER_FALLBACK=false` 并配置真实用户
- **切勿** 在公开网络中将 `TRUST_USER_ID_HEADER` 设为 `true`
- 数据库连接串中的密码应使用强密码
- API 密钥（LLM Provider）建议使用环境变量注入，而非硬编码
