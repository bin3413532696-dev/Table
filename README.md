# 项目用户使用手册

本文档面向第一次接触本项目的用户编写。

你不需要懂“知识库”“本体”“图谱”“断言”这些技术词，也可以正常使用本项目。你可以把它理解成：

- 这是一个个人工作台
- 它可以管理任务
- 它可以记录收入和支出
- 它可以把零散说明、经验、文档、备注整理成一个可搜索的“项目知识中心”
- 它还带有一个智能助手，可以帮你查询和整理内容

---

## 1. 这个项目可以做什么

项目主要包含 6 个页面：

1. `首页`
   用来快速查看今天的工作情况、待办任务和费用概览。

2. `知识库`
   用来集中存放项目说明、业务背景、来源文档、处理记录和关联信息。

3. `任务`
   用来新增、编辑、完成、筛选和删除任务。

4. `费用统计`
   用来记录收入和支出，并查看统计图和导出 CSV。

5. `工具`
   提供计算器、取色器、JSON 格式化等小工具。

6. `设置`
   用来管理个人资料、密码保护、数据导入导出、API 配置。

---

## 2. 如何打开项目

如果项目还没有启动，请在项目目录执行：

```bash
cmd /c npm run dev
```

启动后，在浏览器打开：

```text
http://localhost:3266
```

如果你已经启用了访问密码，打开后会先看到密码锁页面，输入之前设置的数字密码即可进入系统。

---

## 3. 第一次使用时，建议先做什么

第一次进入项目，建议按下面顺序操作：

1. 先进入 `设置`
2. 填写你的昵称、邮箱、个人简介
3. 如果需要，开启访问密码
4. 如果要使用智能助手，先在 `设置 -> API 配置` 中配置可用模型
5. 再进入 `任务` 和 `费用统计`，录入基础数据
6. 最后进入 `知识库`，补充项目说明、业务背景和来源文档

这样做的好处是：

- 任务和费用会自动形成部分知识映射
- 后续在知识库里更容易查到上下文
- 智能助手也更容易理解你的项目内容

---

## 4. 项目里的几个核心概念

为了后面容易理解，先用最简单的话解释几个概念。

### 4.1 任务

任务就是你要做的事，例如：

- 修改页面
- 跟进需求
- 修复问题
- 准备交付

### 4.2 费用记录

费用记录就是收入和支出，例如：

- API 调用费用
- 模型订阅费用
- 服务收入
- 其他支出

### 4.3 知识库

知识库不是“技术人专用的数据库”。

在本项目里，你可以把知识库理解成：

`一个可以搜索的项目资料中心`

它适合存这些内容：

- 项目规则
- 模块说明
- 需求背景
- 会议纪要
- 处理结论
- 交接说明
- 某条任务为什么这样做
- 某笔费用对应什么业务背景

### 4.4 来源文档

来源文档就是“你这条知识是从哪里来的”。

例如：

- 会议纪要
- 需求文档
- 设计稿说明
- 交接文档
- 处理记录

### 4.5 知识条目

知识条目可以理解成“一条被整理好的知识卡片”。

例如：

- “知识库页面收敛方案”
- “费用统计页面数据口径”
- “任务自动映射规则”

---

## 5. 日常推荐使用流程

如果你每天都要用这个项目，建议按下面流程来：

1. 打开 `首页`
   看今天待办和费用概况

2. 进入 `任务`
   新建、更新、完成当天任务

3. 进入 `费用统计`
   记录当天收入和支出

4. 进入 `知识库`
   把当天的重要背景、处理记录、决定依据补充进去

5. 如有需要，打开智能助手
   让它帮你查任务、查费用、查知识说明

## 6. 首页怎么用

首页适合快速总览，不适合做复杂编辑。

你在首页可以看到：

- 当前待办任务数量
- 当前净收益
- 今天日期
- 最近待办任务
- 费用概览

首页每一块卡片都可以点进去，直接进入对应页面。

适合的使用方式：

- 早上打开首页，快速看今天还有哪些任务
- 看一下目前收入、支出、净收益
- 从首页直接跳转到任务页或费用页继续处理

---

## 7. 任务页面怎么用

`任务` 页面用于管理所有待办事项。

### 7.1 可以做什么

- 新增任务
- 设定优先级
- 设置截止日期
- 标记完成
- 编辑任务
- 删除任务
- 按状态筛选
- 按标题搜索
- 批量操作

### 7.2 如何新增任务

常见步骤：

1. 输入任务标题
2. 选择优先级：低 / 中 / 高
3. 如有需要，设置截止日期
4. 保存

建议任务标题写具体，不要太泛。

不推荐：

- “处理一下”
- “看一下这个”

推荐：

- “整理知识库页面的使用文档”
- “核对 5 月 API 支出记录”

### 7.3 如何管理任务

- `全部`：查看所有任务
- `待办`：只看还没完成的任务
- `已完成`：只看已经做完的任务

你也可以按以下方式排序：

- 创建时间
- 优先级
- 截止日期
- 标题

### 7.4 使用建议

- 每个任务尽量只做一件事
- 紧急工作用高优先级
- 能写截止日期的尽量写
- 完成后及时勾选，避免列表失真

---

## 8. 费用统计页面怎么用

`费用统计` 页面用于记录项目相关的收入和支出。

### 8.1 可以做什么

- 录入收入
- 录入支出
- 查看净收益
- 按日期筛选
- 按模型筛选
- 查看月度趋势图
- 查看分类和模型分布
- 导出 CSV

### 8.2 如何新增费用记录

录入时通常会填写：

- 类型：收入 / 支出
- 金额
- 描述
- 分类
- 日期
- 模型

例如：

- 类型：支出
- 金额：120
- 描述：5 月 OpenAI API 调用
- 分类：API 调用费用
- 日期：2026-05-04
- 模型：GPT-4

### 8.3 如何看数据

页面会自动显示：

- 总收入
- 总支出
- 净收益
- 月度收支趋势
- 不同模型的费用占比

### 8.4 导出 CSV 有什么用

如果你想把费用拿去做外部汇报、发给别人、或者用 Excel 再处理，可以直接导出 CSV。

### 8.5 使用建议

- 每条记录的描述尽量写清楚
- 收入和支出不要混写
- 模型字段尽量统一，例如都写 `GPT-4`，不要一会儿写 `gpt4` 一会儿写 `GPT4`

## 9. 知识库页面怎么用

这是本项目最容易让新用户紧张的页面，但你可以把它理解得非常简单：

`知识库 = 一个可以搜索、阅读、补充说明的项目资料中心`

你不需要先理解底层结构，只需要先会用“阅读模式”和“补充知识”。

### 9.1 知识库适合放什么

最适合放以下内容：

- 项目背景
- 模块说明
- 需求上下文
- 交接说明
- 会议纪要
- 业务备注
- 某条任务为什么这样处理
- 某笔费用对应什么背景

### 9.2 不适合放什么

不建议把知识库当成：

- 临时聊天窗口
- 随手记草稿箱
- 完全没有主题的杂乱备注区

如果内容很零碎、很短期、很快就过时，先放任务或临时笔记更合适。

---

## 10. 知识库的三种模式

知识库有三种模式。

### 10.1 阅读模式

这是默认模式，也是最推荐普通用户使用的模式。

适合做这些事：

- 搜索资料
- 阅读知识说明
- 查看来源文档
- 查看最近更新
- 从推荐阅读里快速进入重点内容

如果你只是想“找到资料并看懂”，基本只需要这个模式。

### 10.2 维护模式

适合补充和修正内容。

适合做这些事：

- 编辑知识条目
- 编辑来源文档
- 补充结构关联
- 补充业务备注

如果你已经知道这条知识要放在哪里、该怎么命名，可以进入维护模式。

### 10.3 高级模式

高级模式主要给需要做底层维护和排查的人使用。

里面会看到一些更技术化的内容，例如：

- 结构关联
- 事实记录
- 调试信息

如果你只是正常使用项目，大多数情况下不需要进入高级模式。

## 11. 知识库页面的最简单用法

如果你完全不懂知识库，请按这个最简单流程使用。

### 11.1 先查找

进入 `知识库` 后，先用搜索框。

你可以搜索：

- 任务标题中的关键词
- 业务名称
- 模块名称
- 标签
- 文档标题

例如：

- “费用”
- “交接”
- “任务映射”
- “统计”

### 11.2 再看左侧几个区域

知识库首页通常有这些入口：

- `知识首页`
- `推荐阅读`
- `搜索结果`
- `最近更新`

理解方式：

- `知识首页`：常用入口
- `推荐阅读`：优先建议先看的内容
- `搜索结果`：你当前搜到的内容
- `最近更新`：最近新增或修改过的内容

### 11.3 再看右侧详情

点开任意一项后，右侧通常会显示：

- 标题
- 摘要
- 关键信息
- 来源文档
- 相关条目

你只需要先关注这 4 类信息：

1. 这是什么
2. 它从哪里来
3. 它和什么有关
4. 最近有没有更新

## 12. 知识库里最常见的三种新增方式

如果你要往知识库里加内容，通常只需要用这三种方式。

### 12.1 补充项目知识

什么时候用：

- 你要记录一个长期有效的说明
- 你要解释一个模块、规则、概念
- 你希望别人以后可以搜索到这条内容

适合的例子：

- “知识库页面收敛策略”
- “任务与知识库自动映射规则”
- “费用统计口径说明”

### 12.2 补充说明文档

什么时候用：

- 你手上有一份文档、纪要、说明材料
- 你想把它保存为知识来源
- 你希望后续知识能追溯到这份资料

适合的例子：

- 会议纪要
- 需求讨论摘要
- 交接说明
- 设计约束说明

### 12.3 补充业务备注

什么时候用：

- 你今天处理了一件事，需要补一条背景说明
- 某条任务有特殊原因，需要记录
- 某笔费用背后有业务背景

适合的例子：

- “该任务延期原因说明”
- “本月模型支出上升原因”
- “本轮页面收敛的决策依据”

## 13. 如何新增一条知识

最推荐的做法：

1. 进入 `知识库`
2. 切到 `维护模式`
3. 点击 `补充知识`
4. 选择最符合当前内容的类型
5. 填写标题、摘要、标签、来源
6. 如果是说明文档，再补正文内容和关联条目

### 13.1 标题怎么写

标题要让别人一眼知道这条内容讲什么。

推荐：

- “费用统计页面数据来源说明”
- “任务同步失败排查记录”
- “交接文档修订原则”

不推荐：

- “记录一下”
- “说明”
- “新想法”

### 13.2 摘要怎么写

摘要只需要回答一句话：

`这条内容的核心意思是什么`

### 13.3 标签怎么写

标签用来帮助搜索和分类。

推荐做法：

- 一个内容配 2 到 5 个标签
- 标签尽量短
- 标签命名尽量稳定

例如：

- `handoff`
- `knowledge`
- `finance`
- `task`
- `review`

### 13.4 来源怎么写

来源用于说明“这条内容从哪来”。

例如：

- 交接文档
- 会议纪要
- 需求讨论
- 当日处理记录

---

## 14. 如何判断该放“项目知识”还是“说明文档”

可以用一句话区分：

- `项目知识`：偏整理后的结论
- `说明文档`：偏原始材料或原文内容

举例：

- “本项目数据同步机制说明” 更像项目知识
- “2026-05-04 需求讨论纪要” 更像说明文档

如果你还是拿不准，可以这样做：

1. 先把原始材料放成 `说明文档`
2. 再从中提炼一条 `项目知识`

这样后续搜索和追溯都会更清楚。

## 15. 任务和费用为什么会出现在知识库里

这是本项目的一个特点。

当你在 `任务` 页面和 `费用统计` 页面里录入数据后，系统会自动生成一部分知识映射。你可以理解成：

- 任务不只是任务列表里的一个条目
- 它也会变成知识库中可搜索的上下文

同样：

- 一笔费用不只是财务记录
- 它也可以进入知识库，帮助理解业务背景

这意味着：

- 你以后搜索某个任务名称，可能会在知识库里找到它
- 你搜索某类费用，也可能看到它对应的说明和上下文

## 16. 智能助手怎么用

右下角通常有一个智能助手入口。

### 16.1 它可以帮你做什么

- 查询任务
- 查询费用
- 查询知识内容
- 帮你整理信息
- 给出下一步建议

### 16.2 使用前要注意什么

如果助手显示未连接，通常说明还没有配置可用模型。

请先去：

`设置 -> API 配置`

配置好可用的服务商和模型后，再回来使用。

### 16.3 推荐提问方式

尽量说清楚你要什么，不要只说“帮我看看”。

推荐提问：

- “帮我查询本月费用支出”
- “列出所有未完成任务”
- “知识库里有没有关于交接文档的说明”
- “帮我总结最近更新的知识内容”

### 16.4 遇到确认框是什么意思

如果助手要执行删除、修改等动作，可能会弹出确认操作。

你需要认真看说明，再决定：

- `确认执行`
- `取消`

如果你不确定，就先取消。

## 17. 设置页面怎么用

`设置` 页面主要分为 4 部分。

### 17.1 个人资料

可以设置：

- 昵称
- 邮箱
- 个人简介

这些信息主要用于页面显示和个人标识。

### 17.2 安全隐私

可以设置：

- 访问密码
- 深色模式 / 浅色模式

如果你担心别人直接打开页面看到内容，可以启用数字密码。

### 17.3 数据管理

这是非常重要的一部分。

可以做这些事：

- 导出备份
- 导入备份
- 清空所有数据

强烈建议：

- 在做大改动前先导出备份
- 在交接前导出一次备份
- 在长期使用过程中定期导出

### 17.4 API 配置

如果你要使用智能助手，这里必须配置。

通常需要填写：

- 服务商
- 接口地址
- API Key
- 模型名称

如果不配置，智能助手通常无法正常工作。

## 18. 工具页面怎么用

工具页是辅助页面，目前主要包含：

- 计算器
- 取色器
- JSON 格式化

适合在工作过程中临时使用，不承担项目主数据管理职责。

## 19. 数据备份和恢复建议

为了避免误删、浏览器清理、设备变更带来的损失，建议养成以下习惯：

### 19.1 什么时候导出备份

- 每周至少一次
- 做大批量修改前
- 导入外部数据前
- 交接给别人前

### 19.2 如何导出

1. 进入 `设置`
2. 打开 `数据管理`
3. 点击导出
4. 保存生成的备份文件

### 19.3 如何恢复

1. 进入 `设置`
2. 打开 `数据管理`
3. 选择导入备份文件
4. 等待页面自动刷新

导入前最好先备份当前数据，避免覆盖后无法回退。

## 20. 推荐的知识库整理习惯

如果你希望知识库越用越清晰，建议遵守下面这些规则。

### 20.1 一条知识只讲一个主题

不要把很多不相关的内容塞进同一条里。

### 20.2 原始材料和整理结论分开

推荐结构：

- 原始内容放到 `说明文档`
- 总结后的结论放到 `项目知识`

### 20.3 重要内容一定写来源

如果没有来源，后续别人很难判断内容是否可靠。

### 20.4 标题尽量稳定

标题频繁变来变去，会影响搜索和理解。

### 20.5 不确定就先写摘要

如果你来不及做复杂整理，至少先补一条摘要和来源，后面再慢慢细化。

## 21. 常见使用场景示例

### 场景 1：我今天处理了一条任务

推荐做法：

1. 去 `任务` 页面更新状态
2. 如果过程里有重要背景，去 `知识库`
3. 用 `补充业务备注` 写下原因、处理过程和结论

### 场景 2：我今天有一笔新的 API 支出

推荐做法：

1. 去 `费用统计` 新增一条支出
2. 如果这笔支出有特殊背景，再去 `知识库`
3. 补一条业务备注或说明文档

### 场景 3：我要把项目交给别人

推荐做法：

1. 补齐关键任务说明
2. 补齐关键费用口径说明
3. 在 `知识库` 补充交接背景、注意事项、常见问题
4. 去 `设置 -> 数据管理` 导出备份

### 场景 4：我只想快速找资料

推荐做法：

1. 进入 `知识库`
2. 保持 `阅读模式`
3. 先用搜索框
4. 再看推荐阅读和最近更新

## 22. 常见问题

### 22.1 我不懂知识库，能不能只用任务和费用？

可以。

即使你完全不主动维护知识库，任务和费用页面也可以独立使用。

不过如果你希望后续更方便检索背景、做交接、给智能助手提供上下文，还是建议逐步使用知识库。

### 22.2 为什么知识库里会出现我没有手动创建的内容？

因为任务和费用会自动生成一部分知识映射，这是正常现象。

### 22.3 我应该优先用哪个模式？

普通用户优先用：

- `阅读模式`
- `维护模式`

只有在你明确知道自己要做底层结构维护时，才使用 `高级模式`。

### 22.4 导入失败怎么办？

先检查：

1. 文件是不是本项目导出的备份
2. 文件内容是不是被手动改坏了
3. 文件格式是不是 `.json`

如果仍失败，建议先保留原文件，不要覆盖，找开发人员进一步检查。

### 22.5 清空数据会发生什么？

会删除当前项目里的主要数据内容，通常无法直接撤销。

执行前一定先导出备份。

### 22.6 智能助手为什么不能用？

常见原因：

- 没有配置 API
- 模型不可用
- 当前连接失败

先去 `设置 -> API 配置` 检查。

---

## 23. 给普通用户的最后建议

如果你只记住三件事，请记住这三件：

1. `任务` 管你要做什么
2. `费用统计` 管花了多少钱、赚了多少钱
3. `知识库` 管为什么这么做、资料从哪里来、以后怎么查

最简单、最稳妥的使用方式是：

- 每天更新任务
- 有费用就记账
- 有重要背景就补充到知识库
- 定期导出备份

这样这个项目就会越来越像一个真正能积累经验、方便交接、支持智能助手的个人工作台。
