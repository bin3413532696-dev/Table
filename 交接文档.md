## 1. 项目概述

个人工作站是一个**中文界面的单页应用**（SPA），面向个人效率场景。所有数据存储于浏览器 `localStorage`，并自动同步到项目 `data/` 目录（通过 Webpack Dev Server Middleware），**无独立后端服务**。应用基于 HashRouter 实现 5 个功能页面，共享统一的侧边栏 + 顶栏布局。

### 核心功能模块

| 模块 | 说明 |
|---|---|
| **仪表盘** | 聚合任务、财务统计数据的首页 |
| **任务** | 待办事项管理，优先级和截止日期 |
| **财务** | 收支记录与统计（面向 LLM API 费用场景） |
| **工具** | 计算器、取色器、JSON 格式化工具 |
| **设置** | 个人资料、安全隐私、数据导入导出、主题切换 |

### Dev Server Middleware (auto-sync)

Webpack Dev Server 通过 `setupMiddlewares` 注册了 `POST /api/sync-data` 端点：

- **触发时机**：数据层每次写操作（add/update/delete）后，经 1.5s 防抖，自动 POST 全量数据到 `/api/sync-data`
- **写入逻辑**（`writeSyncData()` 函数）：
  - 任务数据写入 `data/tasks/tasks.json`
  - 财务数据写入 `data/finance/finance.json`
- **生产构建**：auto-sync 仅在 `npm run dev` 开发服务器中生效；生产构建不包含此功能

## 5. 数据层 (localStorage)

**文件**: `src/db/index.ts`

### 数据结构

```typescript
// 财务记录
interface FinanceRecord {
  id: string;           // 时间戳-base36 + 随机数
  type: 'income' | 'expense';
  amount: number;
  description: string;
  category: string;
  date: string;         // ISO 日期
  model?: string;       // 仅支出有模型归属
}

// 任务
interface Task {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;    // ISO 日期
  priority: 'low' | 'medium' | 'high';
  dueDate?: string;
}
```

### 存储键与集合

| localStorage Key | 集合类型 | 操作对象 |
|---|---|---|
| `finance_records` | `FinanceRecord[]` | `financeDB` |
| `tasks_records` | `Task[]` | `taskDB` |

### 用户设置存储

| localStorage Key | 说明 |
|---|---|
| `user_profile` | 用户资料 JSON（name, email, bio） |
| `theme` | 主题设置（'light' / 'dark'） |
| `security_pin_hashed` | PIN 码 SHA-256 哈希值 |

### 导出的 API

**`initDB()`** — 从 localStorage 重新加载所有集合。

**`financeDB`**
- `getAll()` → 按日期降序返回
- `add(record)` / `update(id, partial)` / `delete(id)`
- `getStats()` → `{ income, expense, profit }`
- `getModelStats()` → 按 model 分组的收支汇总

**`taskDB`**
- `getAll()` / `add()` / `update()` / `delete()`
- `toggle(id)` — 翻转 `completed` 状态
- `getStats()` → `{ total, completed, pending }`

**`dataManager`**
- `exportAll()` → JSON 字符串（包含 2 个集合 + 用户设置 + `version: 2` + 导出时间戳）
- `importAll(jsonString)` → 解析并校验，替换集合和用户设置
- `clearAll()` → 清空所有集合和用户设置
- `getStats()` → `{ finance, tasks, totalSize }`

**`createUseDB(React)`** — 创建数据获取 hook 的工厂函数
- 返回 `useDB(fetcher, dependencies)` hook
- 自动订阅数据变更，支持 loading 状态
- 参数：`fetcher` 数据获取函数，`dependencies` 监听的数据集合

### 安全相关 API

**`hashPin(pin: string)`** — 使用 SHA-256 对 PIN 码进行哈希
**`verifyPin(pin: string, hashedPin: string)`** — 验证 PIN 码是否正确

### 重要架构说明

1. **不使用任何外部数据库** — 所有数据存储于 localStorage
2. **数据封装** — Store 类封装数据，外部通过 API 访问，不暴露内部引用
3. **自动同步（auto-sync）** — 每次写操作后触发 1.5s 防抖，通过 `fetch()` POST 到 `http://localhost:3266/api/sync-data`
4. **事务支持** — 提供 `runInTransaction()` 函数，支持多操作原子性
5. **变更订阅** — 提供 `subscribe()` 函数，可监听数据变更
6. **PIN 安全** — PIN 码使用 SHA-256 哈希存储，不明文保存

## 6. 路由架构

**文件**: `src/App.tsx`

使用 `HashRouter`，所有路由是 `Layout` 的子路由：

| 路径 | 组件 | 说明 |
|---|---|---|
| `/` | — | 重定向到 `/dashboard` |
| `/dashboard` | Dashboard | 首页仪表盘（懒加载） |
| `/tasks` | Tasks | 任务管理（懒加载） |
| `/tools` | Tools | 工具集（懒加载） |
| `/finance` | Finance | 财务统计（懒加载） |
| `/settings` | Settings | 设置页面（懒加载） |

### PIN 锁定流程

1. 应用启动时检查 `localStorage.getItem('security_pin_hashed')`
2. 如果存在哈希 PIN，显示 `PinLock` 组件
3. 用户输入 PIN 后，调用 `verifyPin()` 验证
4. 验证成功后显示主应用

- **ThemeContext** (`src/contexts/ThemeContext.tsx`)：提供 `theme`、`setTheme`、`toggleTheme`
- **设置页面**：安全隐私标签页可切换浅色/深色主题
- **持久化**：主题偏好存储于 localStorage (`theme` key)
- **Tailwind 暗色模式**：使用 `class` 策略，所有组件使用 `dark:` 前缀适配

## 10. 代码分割

### 概述

应用使用 React.lazy + Suspense 实现**路由级代码分割**，首屏只加载必要的代码，其他页面在访问时才加载。

### 实现方式

```typescript
// src/App.tsx
import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Tasks = lazy(() => import('./pages/Tasks'));
const Finance = lazy(() => import('./pages/Finance'));
const Tools = lazy(() => import('./pages/Tools'));
const Settings = lazy(() => import('./pages/Settings'));

// 使用 Suspense 包裹路由
<Suspense fallback={<Loading />}>
  <Routes>
    <Route path="dashboard" element={<Dashboard />} />
    {/* ... */}
  </Routes>
</Suspense>
```

### 构建产物

| 文件 | 大小 | 说明 |
|------|------|------|
| bundle.js | ~368 KiB | 主入口（下降 57%） |
| vendors~ | ~398 KiB | 第三方库 |
| Tools chunk | ~38 KiB | 工具页 |
| Finance chunk | ~29 KiB | 财务页 |
| Tasks chunk | ~24 KiB | 任务页 |
| Dashboard chunk | ~15 KiB | 仪表盘 |
| Settings chunk | ~12 KiB | 设置页 |

### 优势

- **首屏加载更快**：主 bundle 从 855 KiB 降到 368 KiB
- **按需加载**：用户访问时才加载对应页面的代码
- **更好的缓存**：每个 chunk 独立缓存，页面更新不影响其他 chunk
