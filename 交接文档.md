## 1. 项目概述

个人工作站是一个**中文界面的单页应用**（SPA），面向个人效率场景。所有数据存储于浏览器 `localStorage`，并自动同步到项目 `data/` 目录（通过 Webpack Dev Server Middleware），**无独立后端服务**。应用基于 HashRouter 实现 6 个功能页面，共享统一的侧边栏 + 顶栏布局。

### 核心功能模块

| 模块 | 说明 |
|---|---|
| **仪表盘** | 聚合任务、笔记、财务统计数据的首页 |
| **笔记** | Markdown 编辑器，支持文件夹/标签/双向链接/图谱视图 |
| **任务** | 待办事项管理，优先级和截止日期 |
| **财务** | 收支记录与统计（面向 LLM API 费用场景） |
| **工具** | 计算器、取色器、JSON 格式化工具 |
| **设置** | 个人资料、通知、安全、数据导入导出 |

### Dev Server Middleware (auto-sync)

Webpack Dev Server 通过 `setupMiddlewares` 注册了 `POST /api/sync-data` 端点：

- **触发时机**：数据层每次写操作（add/update/delete）后，经 1.5s 防抖，自动 POST 全量数据到 `/api/sync-data`
- **写入逻辑**（`writeSyncData()` 函数）：
  - 笔记按文件夹分组，每篇笔记写入 `data/notes/<文件夹名>/<标题>.md`，含 YAML front matter（id, tags, links, createdAt, updatedAt）
  - 任务数据写入 `data/tasks/tasks.json`
  - 财务数据写入 `data/finance/finance.json`
  - 文件夹结构写入 `data/folders/folders.json`
- **生产构建**：auto-sync 仅在 `npm run dev` 开发服务器中生效；生产构建 (`dist/bundle.js`) 不包含此功能

## 5. 数据层 (localStorage)

**文件**: `src/db/index.ts`（约 380 行，这是唯一的数据库文件）

### 数据结构

```typescript
// 财务记录
interface FinanceRecord {
  id: string;           // Date.now().toString()
  type: 'income' | 'expense';
  amount: number;
  description: string;
  category: string;
  date: string;         // ISO 日期
  model?: string;       // 仅支出有模型归属
}

// 任务
interface Task {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;    // ISO 日期
  priority: 'low' | 'medium' | 'high';
  dueDate?: string;
}

// 文件夹
interface Folder {
  id: string;
  name: string;
  parentId: string | null;
  createdAt: string;
  updatedAt: string;
}

// 笔记
interface Note {
  id: string;
  title: string;
  content: string;        // Markdown
  folderId: string | null;
  createdAt: string;
  updatedAt: string;
  tags: string[];
  links: string[];        // 出链笔记标题列表
  backlinks: string[];    // 入链笔记标题列表
}
```

### 存储键与集合

| localStorage Key | 集合类型 | 操作对象 |
|---|---|---|
| `finance_records` | `FinanceRecord[]` | `financeDB` |
| `tasks_records` | `Task[]` | `taskDB` |
| `notes_records` | `Note[]` | `noteDB` |
| `folders_records` | `Folder[]` | `folderDB` |

### 导出的 API

**`initDB()`** — 从 localStorage 重新加载所有集合。

**`financeDB`**
- `getAll()` → 按日期降序返回
- `add(record)` / `update(id, partial)` / `delete(id)`
- `getStats()` → `{ income, expense, profit }`
- `getModelStats()` → 按 model 分组的收支汇总

**`taskDB`**
- `getAll()` / `add()` / `update()` / `delete()`
- `toggle(id)` — 翻转 `completed` 状态
- `getStats()` → `{ total, completed, pending }`

**`noteDB`**
- `getAll()` / `getByFolder(folderId)` / `add()` / `update()` / `delete()`
- `search(query)` — 对 title 和 content 做大小写不敏感匹配
- `updateLinks(noteId, links[])` — 维护双向链接：新链接添加反向引用，移除的链接清理反向引用
- `getBacklinks(noteTitle)` → 返回所有指向该标题的笔记

**`folderDB`**
- `getAll()` / `getByParent(parentId)` / `add()` / `update()` / `delete()`
- `getTree()` → 根文件夹 + `children` 子文件夹（仅一层嵌套）
- 删除文件夹时，其下笔记的 `folderId` 会被置为 null；递归删除子文件夹

**`dataManager`**
- `exportAll()` → JSON 字符串（包含所有 4 个集合 + `version: 1` + 导出时间戳）
- `importAll(jsonString)` → 解析并校验（`Array.isArray()`），替换集合
- `clearAll()` → 清空所有集合
- `getStats()` → `{ finance, tasks, notes, folders, totalSize }`

### 重要架构说明

1. **不使用任何外部数据库** — `@supabase/supabase-js` 和 `sql.js` 虽然安装在 `package.json` 中，但未被引入任何源码文件
2. **所有操作是同步的** — 读写 localStorage 不需要异步
3. **数据初始化** — 模块加载时从 localStorage 读取到内存数组，CRUD 操作同时修改数组和持久化
4. **自动同步（auto-sync）** — 每次写操作后触发 1.5s 防抖，通过 `fetch()` POST 到 `http://localhost:3266/api/sync-data`，开发服务器将数据写入 `data/` 目录。生产构建不含此功能。

### 自动同步机制

每次 `saveToStorage()` 调用（即任何 add/update/delete 操作）都会触发 `scheduleSync()`：

```
写操作 → saveToStorage() → scheduleSync() → 1.5s 防抖 → fetch POST /api/sync-data
```

- 防抖确保连续操作不会产生频繁写入
- 同步仅发送到 `localhost:3266`（开发服务器），失败时静默忽略
- `dataManager.importAll()` 和 `clearAll()` 也会触发同步

## 8. 页面详解

### 8.1 仪表盘 (Dashboard)

**文件**: `src/pages/Dashboard/index.tsx`

**数据源**: `financeDB`, `taskDB`, `noteDB`

**布局**:
- 标题区域（带日期）
- 4 个快速操作卡片（grid 4 列）：待办任务数、笔记数、净收益、今日日期
- 待办任务面板（span 2 列）：最多 3 个
- 最近笔记面板（1 列）：最多 3 个
- 费用概览面板（全宽）：收入/支出/净收益三栏

**状态处理**:
- **空状态**: 任务列表和笔记列表各自有空状态提示
- **无加载状态**: 无需异步加载

### 8.2 笔记 (Notes) — 核心复杂度模块

**文件**: `src/pages/Notes/index.tsx` + 6 个子组件

**主页面 (`index.tsx`)**:
- 三栏布局（可拖拽调整宽度，可折叠）
- 左侧边栏：`FileTree` 组件
- 中间：编辑器或空状态
- 右侧边栏：`Backlinks` / `TagCloud` / `GraphView` 三 Tab 切换
- 拖拽分隔条通过 `mousedown/mousemove/mouseup` 实现，悬停变蓝
- `Ctrl/Cmd+K` 快捷键打开 `CommandPalette`
- 工具栏使用毛玻璃效果（`bg-white/80 backdrop-blur-xl`），分段控件活跃态使用 `bg-white shadow-sm`

**Editor** (`Editor.tsx`):
- CodeMirror 6 引擎，支持 Markdown 语法高亮
- 三种模式：编辑 / 分屏（50/50）/ 预览（全宽 Markdown 渲染）
- 格式化工具栏：粗体、斜体、删除线、标题、列表、引用、代码块、链接、任务列表
- 工具栏使用毛玻璃效果（`bg-white/80 backdrop-blur-xl`），按钮 `rounded-md hover:bg-gray-100`
- 模式切换使用 `bg-gray-100 p-0.5` 容器，活跃态 `bg-white shadow-sm`
- Vim 模式切换（`@replit/codemirror-vim`）
- 全屏切换
- Markdown 预览使用 `react-markdown` + `remark-gfm`，内联 CSS 样式

**FileTree** (`FileTree.tsx`):
- 递归 TreeNode 组件渲染嵌套文件夹
- 展开/折叠动画（Framer Motion AnimatePresence）
- 右键菜单：新建笔记、新建文件夹、重命名、删除（`shadow-xl ring-1 ring-black/5`）
- 内联重命名（双击触发）
- 根级"全部笔记"条目显示总数
- 选中态：`bg-blue-50 text-blue-700`，左侧蓝色指示条（`w-0.5 rounded-full bg-blue-500`）

**Backlinks** (`Backlinks.tsx`):
- 出链：当前笔记 `links` 中引用的笔记
- 反向链接：引用当前笔记标题的其他笔记
- 点击跳转到对应笔记
- 右侧面板统一使用 `text-xs font-semibold uppercase tracking-wider text-gray-500` 标题

**TagCloud** (`TagCloud.tsx`):
- 统计所有笔记中的标签频率
- 按频率降序排列，字体大小 12px-20px 归一化
- 点击切换筛选

**GraphView** (`GraphView.tsx`):
- D3 v7 力导向图
- 节点半径：当前笔记 25px，其他 15px + 链接数 * 2
- 力模拟参数：`forceLink(80)`, `forceManyBody(-200)`, `forceCollide(radius+5)`
- 节点可拖拽，点击跳转笔记
- 空状态显示提示

**CommandPalette** (`CommandPalette.tsx`):
- 模态覆盖层，搜索输入框
- 三种结果类型：动作（新建笔记）、笔记、文件夹
- 键盘导航：上下箭头、Enter 确认、Escape 关闭
- Fuse.js 模糊搜索

### 8.3 任务 (Tasks)

**文件**: `src/pages/Tasks/index.tsx`

**数据源**: `taskDB`

**功能**:

### 8.4 财务 (Finance)

**文件**: `src/pages/Finance/index.tsx`

**数据源**: `financeDB`

**功能**:
- 三张统计卡片：总收入、总支出、净收益
- 收支记录表格：类型切换、日期/模型筛选、编辑/删除
- 模型统计面板：按模型分组展示
- 添加/编辑表单：模态对话框，动态分类列表（收入/支出不同）

**分类体系**:
- 收入：`['API调用收入', '服务收入', '其他收入']`
- 支出：`['API调用费用', '模型订阅', '基础设施', '其他支出']`
- 模型列表：`['GPT-4', 'GPT-3.5', 'Claude', 'Gemini', '其他']`

**状态处理**: 使用 `dbReady` 标志，加载完成前显示"加载中..."

### 8.5 工具 (Tools)

**文件**: `src/pages/Tools/index.tsx`

三个独立工具：
1. **计算器** — 四则运算，连续计算支持
2. **取色器** — 实时 HEX/RGB 预览 + 预设颜色网格（15 色）
3. **JSON 格式化** — 格式化 / 压缩 / 复制 / 清空，错误提示

无数据持久化。

### 8.6 设置 (Settings)

**文件**: `src/pages/Settings/index.tsx`

左侧边栏 + 右侧内容的标签页布局。

**标签页**:

| 标签 | 功能 |
|---|---|
| profile | 昵称/邮箱/简介，存 `user_profile` 至 localStorage |
| notifications | 4 个切换开关，存 `notification_settings` |
| security | 4-6 位数字 PIN 码，存 `security_pin` |
| data | 数据统计 + 导出 / 导入 / 笔记导出 / 同步状态 |

**数据管理**（紧凑单列布局）:
- 统计行：各集合记录数 + 总大小（内联文本）
- 同步状态条：显示上次同步时间（灰色背景徽章）
- 导出 JSON：`dataManager.exportAll()` → 下载 `workspace_backup_YYYY-MM-DD.json`
- 导入 JSON：文件选择 → `dataManager.importAll()` → 成功/失败反馈
- 导出笔记为 Markdown：遍历所有笔记生成为 `.md` 文件批量下载
- 清除数据：灰色小字链接 → 确认对话框 → `clearAll()` → 页面重载

- `src/hooks/useTheme.ts` 是存根（始终返回 `isDark: false`），主题切换尚未在 React 层实现。设置页面中对应的外观主题选项卡已被移除，目前无 UI 入口配置主题。
- Tailwind 暗色模式使用 `class` 策略

### 自定义滚动条

`.scrollbar-thin` 类提供 Webkit 浏览器的自定义滚动条样式（暗色主题）。

## 10. 已知问题与技术债务

### 数据层问题

1. **主题系统已无 UI 入口**：`src/hooks/useTheme.ts` 始终返回 `isDark: false`。CSS 变量和 HTML 级别的脚本仍支持暗色主题，但设置页中的"外观主题"选项卡已被移除，React 层无切换逻辑且无配置入口。

2. **Folder 嵌套有限**：`folderDB.getTree()` 仅支持一层嵌套（根 + children），不支持深层递归。

3. **Supabase 和 sql.js 未使用**：这两个依赖已安装但未被任何源码引用。

### 类型与代码质量

1. **Editor.tsx insertText 函数**：基于 DOM 选区操作，在 CodeMirror 环境中可能行为异常。推荐使用 CodeMirror 的 `view.dispatch` API 替代。

2. **隐式数据关联**：笔记间的 `[[wikilinks]]` 基于标题字符串匹配，笔记重命名不会更新其他笔记中的链接引用。

3. **无错误边界**：应用未配置 React Error Boundary，组件崩溃会白屏。

### 前端体验

1. **暗色模式不完整**：CSS 变量已定义暗色值，但 React 层无切换机制，设置页中对应的外观主题选项卡已被移除，目前无法通过 UI 切换主题。大部分组件未适配暗色样式。

2. **移动端适配缺失**：应用假设桌面屏幕宽度，侧边栏固定 256px，未做响应式设计。

3. **浏览器数据易丢失**：数据存储于 localStorage，用户清除浏览器缓存会导致数据丢失。auto-sync 仅在开发服务器中生效，生产构建需配合 `scripts/export-to-files.js` 定期备份。

## 11. 常见开发任务

### 新增页面

1. 在 `src/pages/` 下创建目录和组件
2. 在 `src/App.tsx` 添加路由（`<Route path="..." element={<NewPage />} />`）
3. 在 `Sidebar.tsx` 的 `menuItems` 数组中添加菜单项
4. 如需数据持久化，使用 `src/db/index.ts` 中对应集合的 API

### 新增数据集合

1. 在 `src/db/index.ts` 中定义 TypeScript 接口
2. 添加 localStorage key 常量
3. 创建 CRUD 操作对象（参考 `financeDB`/`taskDB` 模式）
4. 在 `initDB()`、`dataManager.exportAll()`/`importAll()`/`clearAll()`/`getStats()` 中注册新集合
5. 在 `CLAUDE.md` 中更新数据层文档

### 修改主题

- 暗色模式切换：实现 `useTheme` hook（当前是存根） → 在 `<html>` 上切换 `.dark` 类 → 验证各组件暗色样式
- 共享的 `_primary` 颜色在 `tailwind.config.js` 的 `theme.extend.colors` 中定义

### 调试提示

- 数据存储在 localStorage，可在浏览器 DevTools → Application → Local Storage 中查看/编辑
- 启动 `npm run dev` 后默认访问 `http://localhost:3266/`
- 类型检查在前，构建在后：`npm run typecheck && npm run build`

### 构建部署

1. `npm run build` → 输出到 `dist/bundle.js`
2. 将 `dist/` 目录部署到静态服务器
3. 注意：应用使用 HashRouter，部署时无需配置 URL 重写规则
