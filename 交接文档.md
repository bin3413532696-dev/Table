## 1. 项目概述

个人工作站是一个**中文界面的单页应用**（SPA），面向个人效率场景。所有数据存储于浏览器 `localStorage`，并自动同步到项目 `data/` 目录（通过 Webpack Dev Server Middleware），**无独立后端服务**。应用基于 HashRouter 实现 6 个功能页面，共享统一的侧边栏 + 顶栏布局。

### 核心功能模块

| 模块 | 说明 |
|---|---|
| **仪表盘** | 聚合任务、笔记、财务统计数据的首页 |
| **笔记** | Markdown 编辑器，支持文件夹/标签/双向链接/图谱视图 |
| **任务** | 待办事项管理，优先级和截止日期 |
| **财务** | 收支记录与统计（面向 LLM API 费用场景） |
| **工具** | 计算器、取色器、JSON 格式化工具 |
| **设置** | 个人资料、通知、安全、数据导入导出、主题切换 |

### Dev Server Middleware (auto-sync)

Webpack Dev Server 通过 `setupMiddlewares` 注册了 `POST /api/sync-data` 端点：

- **触发时机**：数据层每次写操作（add/update/delete）后，经 1.5s 防抖，自动 POST 全量数据到 `/api/sync-data`
- **写入逻辑**（`writeSyncData()` 函数）：
  - 笔记按文件夹分组，每篇笔记写入 `data/notes/<文件夹名>/<标题>.md`，含 YAML front matter（id, tags, links, createdAt, updatedAt）
  - 任务数据写入 `data/tasks/tasks.json`
  - 财务数据写入 `data/finance/finance.json`
  - 文件夹结构写入 `data/folders/folders.json`
- **生产构建**：auto-sync 仅在 `npm run dev` 开发服务器中生效；生产构建 (`dist/bundle.js`) 不包含此功能

## 5. 数据层 (localStorage)

**文件**: `src/db/index.ts`

### 数据结构

```typescript
// 财务记录
interface FinanceRecord {
  id: string;           // Date.now().toString()
  type: 'income' | 'expense';
  amount: number;
  description: string;
  category: string;
  date: string;         // ISO 日期
  model?: string;       // 仅支出有模型归属
}

// 任务
interface Task {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;    // ISO 日期
  priority: 'low' | 'medium' | 'high';
  dueDate?: string;
}

// 文件夹
interface Folder {
  id: string;
  name: string;
  parentId: string | null;
  createdAt: string;
  updatedAt: string;
  noteCount: number;    // 文件夹内笔记数量
}

// 笔记
interface Note {
  id: string;
  title: string;
  content: string;        // Markdown
  folderId: string | null;
  createdAt: string;
  updatedAt: string;
  tags: string[];
  links: string[];        // 出链笔记 ID 列表
  backlinks: string[];    // 入链笔记 ID 列表
}
```

### 存储键与集合

| localStorage Key | 集合类型 | 操作对象 |
|---|---|---|
| `finance_records` | `FinanceRecord[]` | `financeDB` |
| `tasks_records` | `Task[]` | `taskDB` |
| `notes_records` | `Note[]` | `noteDB` |
| `folders_records` | `Folder[]` | `folderDB` |

### 导出的 API

**`initDB()`** — 从 localStorage 重新加载所有集合。

**`financeDB`**
- `getAll()` → 按日期降序返回
- `add(record)` / `update(id, partial)` / `delete(id)`
- `getStats()` → `{ income, expense, profit }`
- `getModelStats()` → 按 model 分组的收支汇总

**`taskDB`**
- `getAll()` / `add()` / `update()` / `delete()`
- `toggle(id)` — 翻转 `completed` 状态
- `getStats()` → `{ total, completed, pending }`

**`noteDB`**
- `getAll()` / `getByFolder(folderId)` / `add()` / `update()` / `delete()`
- `search(query)` — 对 title 和 content 做大小写不敏感匹配
- `updateLinks(noteId, links[])` — 维护双向链接：新链接添加反向引用，移除的链接清理反向引用
- `getBacklinks(noteId)` → 返回所有链接到该笔记的其他笔记

**`folderDB`**
- `getAll()` / `getByParent(parentId)` / `add()` / `update()` / `delete()`
- `getTree()` → 返回文件夹树结构（支持无限层级嵌套）
- 删除文件夹时，其下笔记的 `folderId` 会被置为 null；递归删除子文件夹

**`dataManager`**
- `exportAll()` → JSON 字符串（包含所有 4 个集合 + `version: 1` + 导出时间戳）
- `importAll(jsonString)` → 解析并校验（`Array.isArray()`），替换集合
- `clearAll()` → 清空所有集合
- `getStats()` → `{ finance, tasks, notes, folders, totalSize }`

### 重要架构说明

1. **不使用任何外部数据库** — 所有数据存储于 localStorage
2. **数据封装** — Store 类封装数据，外部通过 API 访问，不暴露内部引用
3. **自动同步（auto-sync）** — 每次写操作后触发 1.5s 防抖，通过 `fetch()` POST 到 `http://localhost:3266/api/sync-data`
4. **事务支持** — 提供 `runInTransaction()` 函数，支持多操作原子性
5. **变更订阅** — 提供 `subscribe()` 函数，可监听数据变更

- **悬停提示**：显示出链数、入链数、标签
- **力模拟参数**：`forceLink(60)`, `forceManyBody(-150)`, `forceCollide(radius+8)`, `forceRadial` 用于孤立节点
- 节点可拖拽，点击跳转笔记
- 正确清理 D3 事件监听器，避免内存泄漏
- 子模块拆分：
  - `types.ts` - 类型定义（GraphNode、GraphLink、COLORS）
  - `useGraphSimulation.ts` - D3 力导向图模拟 hook
  - `useGraphData.ts` - 图谱数据构建 hook
  - `GraphControls.tsx` - 控制面板和统计组件

**CommandPalette** (`CommandPalette.tsx`):
- 模态覆盖层，搜索输入框
- 三种结果类型：动作（新建笔记）、笔记、文件夹

## 11. 常见开发任务

### 新增页面

1. 在 `src/pages/` 下创建目录和组件
2. 在 `src/App.tsx` 添加路由
3. 在 `Sidebar.tsx` 的 `menuItems` 数组中添加菜单项
4. 如需数据持久化，使用 `src/db/index.ts` 中对应集合的 API

### 新增数据集合

1. 在 `src/db/index.ts` 中定义 TypeScript 接口
2. 添加 localStorage key 常量（`STORAGE_KEYS`）
3. 创建验证函数（如 `isValidXxx`）
4. 创建 Store 实例
5. 导出 CRUD 操作对象
6. 在 `dataManager` 中注册新集合
7. 在 `src/store/useStore.tsx` 中添加对应的 actions
8. 在 `交接文档.md` 中更新数据层文档

### 修改主题

- 使用 `useTheme()` hook 获取当前主题
- 组件样式使用 `dark:` 前缀
- CSS 变量使用 `var(--bg-primary)` 等

### 调试提示

- 数据存储在 localStorage，可在浏览器 DevTools → Application → Local Storage 中查看
- 启动 `npm run dev` 后默认访问 `http://localhost:3266/`
- 类型检查：`npm run typecheck`

### 构建部署

1. `npm run build` → 输出到 `dist/bundle.js`
2. 将 `dist/` 目录部署到静态服务器
3. 注意：应用使用 HashRouter，部署时无需配置 URL 重写规则
