### H-01. ~~Provider API Key 纳入配置哈希计算（信息泄露）~~ ✅ 已修复

- **文件**：`server/src/shared/auth.ts` 第 22 行
- **描述**：`computeProviderConfigHash` 计算 `sha256(baseUrl|apiKey|model)`，明文 apiKey 被拼接到哈希输入中。若攻击者获取哈希值且知道 baseUrl 和 model（非秘密），可暴力破解 apiKey。
- **修复**：改为 `sha256(baseUrl | sha256(apiKey) | model)` 或使用 HMAC。
- **已修复内容**：改为 `sha256(baseUrl|sha256(apiKey)|model)`，apiKey 先单独哈希再拼接，无法从配置哈希反推。

- **描述**：每次调用 `getCurrentUserId()`、`resolveRequestUserContext()` 等都重新解析环境变量。高负载下浪费 CPU。
- **修复**：启动时解析一次并缓存为单例。
- **备注**：单用户本机场景请求量极少，环境变量解析开销可忽略。

### M-04. 乐观锁 version 字段在更新时可选 ⏸️ 暂不处理

- **文件**：`server/src/modules/tasks/repository.ts` 第 41-42 行；`server/src/modules/tasks/schema.ts`
- **描述**：`updateTask` 的 `version` 字段为 optional，客户端省略时更新无并发检查，可能丢失更新。
- **修复**：更新操作中 `version` 改为必填。
- **备注**：单用户本机场景无并发冲突，数据覆盖风险极低。

### M-05. 财务更新存在双重获取（TOCTOU） ⏸️ 暂不处理

- **文件**：`server/src/modules/finance/service.ts` 第 26-28 行；`server/src/modules/finance/repository.ts` 第 42-48 行
- **描述**：`updateFinanceRecordEntry` 先 `findFinanceRecordById` 再 `updateFinanceRecord`（内部又做一次 findFirst），存在时间窗口竞态。
- **修复**：移除 service 层冗余查询，使用单次原子更新。
- **备注**：单用户本机场景无并发请求，竞态条件无法触发。

### M-06. 财务记录日期解析未校验 ✅ 已修复

- **文件**：`server/src/modules/finance/repository.ts` 第 25 行
- **描述**：`new Date(input.date ?? input.recordDate ?? '')` 可产生 `Invalid Date`。Zod schema 仅用 `.optional()` 而非日期格式校验。
- **修复**：添加 Zod 日期格式验证 `z.string().regex(/^\d{4}-\d{2}-\d{2}$/)`。
- **已修复内容**：
  - 在 `schema.ts` 中新增 `dateStringSchema`，校验 YYYY-MM-DD 格式
  - 使用 `.refine()` 验证日期值有效性（防止如 2026-99-99）
  - 测试验证：无效日期被拒绝，有效日期通过

### M-07. Provider 更新时清空 API Key 被静默忽略 ⏸️ 暂不处理

- **文件**：`server/src/modules/providers/service.ts` 第 207-209、235-236 行
- **描述**：显式传 `apiKey: ''` 清空密钥时，`normalizeOptionalString` 返回 null，条件判断保留旧密钥。用户意图清空密钥但实际未生效。
- **修复**：区分 `undefined`（未提供）和 `''`（意图清空），空字符串时设置 `apiKeyEncrypted: null`。
- **备注**：UI 行为不一致问题，非安全漏洞。单用户场景影响低。

### M-08. 知识库搜索 tsquery 构建不够健壮 ⏸️ 暂不处理

- **文件**：`server/src/modules/knowledge/repository.ts` 第 128-133 行
- **描述**：手动拼接 tsquery 字符串，虽有基本字符过滤，但特殊 Unicode 或精心构造的输入可能产生异常查询。
- **修复**：使用 `plainto_tsquery` 或 `phraseto_tsquery` 处理不受信输入。
- **备注**：代码健壮性问题，非安全漏洞。单用户不会故意输入恶意内容。

### M-09. Agent 执行迭代次数无上限校验 ⏸️ 暂不处理

- **文件**：`server/src/modules/agent/executor.ts` 第 37 行
- **描述**：`MAX_AGENT_EXECUTION_ITERATIONS = Number(process.env.MAX_AGENT_ITERATIONS) || 5`，若环境变量设为极大值（如 1000），Agent 可循环极长时间。
- **修复**：添加边界检查 `Math.min(Math.max(value, 1), 20)`，移入 config schema 校验。
- **备注**：Agent 相关问题，默认值 5 已足够，用户不会恶意配置极大值。暂不处理。

### M-10. 快照导入先删后插存在风险 ⏸️ 暂不处理

- **文件**：`server/src/modules/maintenance/service.ts` 第 225-244 行
- **描述**：`importBusinessSnapshot` 在事务内先删除所有数据再插入新数据。虽然事务保证原子性，但备份在事务外获取，备份查询失败可能导致数据丢失。
- **修复**：在事务内获取备份，或先验证所有插入数据再执行删除。
- **备注**：单用户手动操作，导入前已有心理预期，风险可控。

### M-11. 种子数据在三处硬编码重复

- **文件**：`server/src/shared/auth.ts` 第 210-295 行；`server/src/modules/maintenance/service.ts` 第 355-444 行；`server/scripts/seed.js` 第 192-235 行
- **描述**：相同的种子数据（任务标题、财务记录、知识笔记、预设标签）在三处独立维护，修改一处容易遗漏其他。
- **修复**：提取到单一共享模块。

### M-12. 无安全响应头（Helmet）

- **文件**：`server/src/app.ts`
- **描述**：未设置 `X-Content-Type-Options`、`X-Frame-Options`、`Strict-Transport-Security`、`Content-Security-Policy` 等安全头。
- **修复**：添加 `@fastify/helmet` 插件。

### M-13. 生产构建未提取 CSS（FOUC）

- **文件**：`webpack.config.js` 第 36-38 行
- **描述**：生产构建使用 `style-loader`（JS 注入 CSS），导致首屏无样式闪烁。
- **修复**：生产模式使用 `MiniCssExtractPlugin.loader` 替代 `style-loader`。

### M-14. 缺少测试框架和 Lint 工具

- **文件**：`package.json`
- **描述**：无 `test`/`lint`/`format` 脚本，无测试框架（vitest/jest），无 ESLint/Prettier。代码质量完全依赖人工审查。
- **修复**：添加 vitest + ESLint + Prettier 及对应脚本。

### M-15. 数据库连接池参数拼接方式脆弱

- **文件**：`server/src/db/client.ts` 第 12-15 行
- **描述**：连接池参数通过 `?connection_limit=20&pool_timeout=10` 拼接到 `DATABASE_URL`。若 URL 已含查询参数（如 `?sslmode=require`），将产生无效 URL。
- **修复**：正确解析 URL 后添加查询参数，或使用 Prisma 连接池配置选项。

### M-16. 健康检查端点不验证数据库连接

- **文件**：`server/src/modules/health/routes.ts` 第 4-11 行
- **描述**：始终返回 `{ ok: true }`，数据库不可用时仍报告健康，误导负载均衡器。
- **修复**：添加 `SELECT 1` 查询验证数据库连接。

### M-17. 软删除策略不一致

- **文件**：`prisma/schema.prisma`
- **描述**：Task/FinanceRecord/ApiProvider/KnowledgeNote 使用软删除（deletedAt），AgentRun/AgentMessage/ToolExecution/KnowledgePresetTag 使用硬删除。维护重置服务对 KnowledgePresetTag 执行硬删除。
- **修复**：统一软删除/硬删除策略并文档化。

### M-18. 前端数据流模式不统一

- **文件**：`src/db/index.ts` vs `src/pages/Knowledge/index.tsx`
- **描述**：Task/Finance 使用 Store + useDB 模式，Knowledge 绕过 Store 直接 API 调用 + useState。架构不一致增加维护成本。
- **修复**：统一数据访问模式。

### M-19. 前端页面缺少独立 Error Boundary

- **文件**：`src/pages/` 所有页面组件
- **描述**：仅有顶层 ErrorBoundary，单个页面错误导致整个应用崩溃。
- **修复**：为每个页面添加独立 Error Boundary。

### M-20. SyncEngine 单例无销毁方法

- **文件**：`src/sync/SyncEngine.ts` 第 44-62 行
- **描述**：单例模式无 `dispose()` 方法，热重载时定时器和监听器可能泄漏。

- **文件**：`server/src/shared/session.ts` 第 56-58 行

- **描述**：不生成请求 ID，难以跨服务关联日志。

- **描述**：返回所有 Provider 无分页限制，理论上可无限创建。
- **修复**：添加分页参数或限制每用户最大 Provider 数。

### L-04. 知识库标签查询 CROSS JOIN LATERAL 性能风险

- **文件**：`server/src/modules/knowledge/repository.ts` 第 214-234 行
- **描述**：`listAllTags` 使用 `CROSS JOIN LATERAL jsonb_array_elements_text`，大量笔记和标签时可能慢查询。
- **修复**：为 `tags_json` 添加 GIN 索引；考虑缓存结果。

### L-05. toStringArray 工具函数重复定义

- **文件**：`server/src/modules/knowledge/dto.ts` 第 7-11 行；`server/src/modules/knowledge/repository.ts` 第 29-33 行
- **描述**：同一函数在两个文件中重复定义。
- **修复**：移至共享工具模块。

### L-06. Webpack 别名引用不存在的包

- **文件**：`webpack.config.js` 第 43-48 行
- **描述**：`@huggingface/transformers`、`sharp`、`onnxruntime-node/web` 别名指向的包不在 dependencies 中。
- **修复**：移除未使用的别名。

### L-07. TypeScript 未启用 noUncheckedIndexedAccess

- **文件**：`tsconfig.json`、`tsconfig.server.json`
- **描述**：数组/对象索引访问返回 `T` 而非 `T | undefined`，可能遗漏空值检查。
- **修复**：添加 `"noUncheckedIndexedAccess": true`。

### L-08. .gitignore 缺少重要模式

- **文件**：`.gitignore`
- **描述**：缺少 `.env.local`、`.env.production`、`*.pem`、`*.key`、`.DS_Store`、`Thumbs.db`、`coverage/` 等模式。
- **修复**：补充完整的忽略规则。

### L-09. date-fns 版本过旧

- **文件**：`package.json` 第 42 行
- **描述**：使用 `date-fns@^2.30.0`，v3 已长期可用且有破坏性变更。
- **修复**：升级到 v3 或评估是否可用原生 Intl API 替代。

### L-10. 加密密钥派生未使用标准 KDF

- **文件**：`server/src/shared/crypto.ts` 第 9-12 行
- **描述**：AES-256 密钥通过简单 SHA-256 哈希派生，无盐值、无迭代次数。低熵密钥时安全性不足。
- **修复**：使用 PBKDF2 或 HKDF 加盐派生。

### L-11. Session Token 无撤销机制

- **文件**：`server/src/shared/session.ts` 第 15-18 行
- **描述**：Token 格式为 `userId.expiresAt.hmacSignature`，无 Token ID，无法撤销已签发的 Token。
- **修复**：添加 Token 黑名单或使用可撤销的 Token 格式。
